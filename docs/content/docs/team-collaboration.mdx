| Role | Level | Scope | Description |
|------|-------|-------|-------------|

## Permission System

Supercheck's RBAC system provides granular control over all platform resources with context-aware permission evaluation.

### Permission Matrix

| Resource | SUPER_ADMIN | ORG_OWNER | ORG_ADMIN | PROJECT_ADMIN | PROJECT_EDITOR | PROJECT_VIEWER |
|----------|-------------|-----------|-----------|---------------|----------------|------------------|
| **Organizations** | ✅ | ✅ | ✅ | ❌ | ❌ | ❌ |
| **Projects** | ✅ | ✅ | ✅ | ✅ | ❌ | ❌ |
| **Tests** | ✅ | ✅ | ✅ | ✅ | ✅ | 👁️ |
| **Jobs** | ✅ | ✅ | ✅ | ✅ | ✅ | 👁️ |

#### Resource Permissions

- **Create**: Add new resources (tests, jobs, monitors, etc.)
- **Read**: View resource details and configurations
- **Update**: Modify existing resources and settings
- **Delete**: Remove resources and associated data
- **Execute**: Run tests and trigger job executions

#### Administrative Permissions

- **User Management**: Create, update, and delete user accounts
- **Role Assignment**: Assign and modify user roles
- **Organization Settings**: Configure organization-level settings
- **Project Management**: Create and manage projects
- **Audit Access**: View audit logs and system activities

### Permission Evaluation

<Callout type="note">
Permissions are evaluated in the current organization/project context, ensuring users only have access to resources they're authorized to manage.
</Callout>

#### Permission Check Implementation

```typescript
// Example permission check function
function hasPermission(
  user: User,
  permission: string,
  resource: Resource,
  context: Context
): boolean {
  const userRole = getUserRole(user, context);
  const rolePermissions = ROLE_PERMISSIONS[userRole];
  
  // Check if role has the required permission
  if (!rolePermissions.includes(permission)) {
    return false;
  }
  
  // Check resource-level permissions
  if (resource.ownerId !== user.id && !rolePermissions.includes('manage-all')) {
    return false;
  }
  
  // Check context-specific permissions
  return checkContextPermissions(user, permission, resource, context);
}

// Example usage
const canEditTest = hasPermission(
  currentUser,
  "test:update",
  testResource,
  projectContext
);

const canDeleteMonitor = hasPermission(
  currentUser,
  "monitor:delete",
  monitorResource,
  projectContext
);
```
