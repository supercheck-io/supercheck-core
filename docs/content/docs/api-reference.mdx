# API Reference

The Supercheck API provides comprehensive programmatic access to all platform features. This reference documentation covers all available endpoints, authentication methods, and usage examples.

<Callout type="tip">
The Supercheck API follows RESTful principles with JSON responses, proper HTTP status codes, and comprehensive error handling. All API requests should be made to `https://api.supercheck.io/v1`.
</Callout>

## API Overview

### Base URL

```
Production: https://api.supercheck.io/v1
Staging: https://api-staging.supercheck.io/v1
Development: http://localhost:3001/api/v1
```

### Authentication

Supercheck API supports multiple authentication methods:

#### API Key Authentication

```http
Authorization: Bearer YOUR_API_KEY
```

#### Session-Based Authentication

```http
Cookie: session_id=your-session-id
```

### Error Handling

The API uses standard HTTP status codes and returns detailed error information:

```json
{
  "error": {
    "code": "VALIDATION_ERROR",
    "message": "Invalid request parameters",
    "details": {
      "field": "title",
      "reason": "Required field is missing"
    }
  },
  "timestamp": "2024-01-15T10:30:00Z",
  "requestId": "req-123456789"
}
```

### Rate Limiting

API requests are rate limited based on your subscription plan:

| Plan | Requests/Minute | Requests/Hour | Concurrent Requests |
|-------|----------------|---------------|-------------------|
| Free | 60 | 1,000 | 5 |
| Pro | 300 | 10,000 | 20 |
| Enterprise | 1,000 | 100,000 | 100 |

Rate limit headers are included in responses:

```http
X-RateLimit-Limit: 300
X-RateLimit-Remaining: 295
X-RateLimit-Reset: 1642694400
```

## Authentication Endpoints

### Generate API Key

Generate a new API key for programmatic access.

**Endpoint:** `POST /api/auth/api-keys`

**Request Body:**
```json
{
  "name": "My API Key",
  "description": "API key for CI/CD integration",
  "scopes": ["tests:read", "tests:write", "jobs:execute"],
  "expiresAt": "2024-12-31T23:59:59Z"
}
```

**Response:**
```json
{
  "id": "key-123456789",
  "name": "My API Key",
  "apiKey": "sk-1234567890abcdef",
  "scopes": ["tests:read", "tests:write", "jobs:execute"],
  "createdAt": "2024-01-15T10:30:00Z",
  "expiresAt": "2024-12-31T23:59:59Z",
  "lastUsedAt": null
}
```

### List API Keys

List all API keys associated with your account.

**Endpoint:** `GET /api/auth/api-keys`

**Response:**
```json
{
  "apiKeys": [
    {
      "id": "key-123456789",
      "name": "My API Key",
      "scopes": ["tests:read", "tests:write", "jobs:execute"],
      "createdAt": "2024-01-15T10:30:00Z",
      "expiresAt": "2024-12-31T23:59:59Z",
      "lastUsedAt": "2024-01-15T11:00:00Z"
    }
  ],
  "pagination": {
    "page": 1,
    "limit": 20,
    "total": 5
  }
}
```

### Revoke API Key

Revoke an API key to disable access.

**Endpoint:** `DELETE /api/auth/api-keys/{keyId}`

**Response:** `204 No Content`

### List Tests

Retrieve a list of all tests in your project.

**Endpoint:** `GET /api/tests`

**Query Parameters:**
- `page` (integer): Page number (default: 1)
- `limit` (integer): Items per page (default: 20)
- `search` (string): Search term

## Error Handling

### Common Error Codes

| Status Code | Description | Solution |
|------------|-------------|----------|
| 400 | Bad Request | Check request format and parameters |
| 401 | Unauthorized | Check API key and authentication |
| 403 | Forbidden | Check user permissions |
| 404 | Not Found | Check resource ID and URL |
| 429 | Too Many Requests | Check rate limits and implement backoff |
| 500 | Internal Server Error | Contact support |

### Error Response Format

```json
{
  "error": {
    "code": "VALIDATION_ERROR",
    "message": "Invalid request parameters",
    "details": {
      "field": "title",
      "reason": "Required field is missing"
    }
  },
  "timestamp": "2024-01-15T10:30:00Z",
  "requestId": "req-123456789"
}
```

### Rate Limit Handling

```javascript
// Handle rate limits with exponential backoff
async function makeAPICall(url, options, retries = 3) {
  try {
    const response = await fetch(url, options);
    
    if (response.status === 429) {
      const resetTime = parseInt(response.headers.get('X-RateLimit-Reset'));
      const waitTime = resetTime - Date.now();
      
      if (retries > 0) {
        await new Promise(resolve => setTimeout(resolve, waitTime));
        return makeAPICall(url, options, retries - 1);
      }
    }
    
    return response;
  } catch (error) {
    if (retries > 0) {
      await new Promise(resolve => setTimeout(resolve, 1000));
      return makeAPICall(url, options, retries - 1);
    }
    throw error;
  }
}
```
