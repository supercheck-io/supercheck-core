# Quick Reference

This quick reference guide provides essential information for using the Supercheck platform efficiently, including keyboard shortcuts, common commands, configuration examples, and troubleshooting tips.

## Keyboard Shortcuts

| Action            | Windows/Linux | macOS     | Description             |
| ----------------- | ------------- | --------- | ----------------------- |
| **Search**        | `Ctrl + F`    | `Cmd + F` | Open global search      |
| **Help**          | `F1`          | `F1`      | Open help documentation |

| **Notifications** | `Ctrl + N`    | `Cmd + N` | Open notifications      |

## Common Commands

### CLI Commands

#### Test Management

```bash
# List all tests
supercheck tests list

# Create new test
supercheck tests create --title "API Test" --type "api"

# Run specific test
supercheck tests run --id "test-123456789"

# Run tests by tag
supercheck tests run --tags "critical,smoke"

# Delete test
supercheck tests delete --id "test-123456789"

# Export tests
supercheck tests export --format "json" --output "tests.json"

# Import tests
supercheck tests import --format "json" --input "tests.json"
```

#### Job Management

```bash
# List all jobs
supercheck jobs list

# Create new job
supercheck jobs create --name "Daily Tests" --tests "test-1,test-2" --schedule "0 2 * * *"

# Run job
supercheck jobs run --id "job-123456789"

# Schedule job
supercheck jobs schedule --id "job-123456789" --cron "0 2 * * *"

# Delete job
supercheck jobs delete --id "job-123456789"

# Get job status
supercheck jobs status --id "job-123456789"
```

#### Monitor Management

```bash
# List all monitors
supercheck monitors list

# Create new monitor
supercheck monitors create --name "API Monitor" --type "http" --target "https://api.example.com/health"

# Test monitor
supercheck monitors test --id "monitor-123456789"

# Enable monitor
supercheck monitors enable --id "monitor-123456789"

# Disable monitor
supercheck monitors disable --id "monitor-123456789"

# Delete monitor
supercheck monitors delete --id "monitor-123456789"
```

#### Variable Management

```bash
# List all variables
supercheck variables list

# Create variable
supercheck variables create --key "API_URL" --value "https://api.example.com"

# Create secret
supercheck variables create --key "API_KEY" --value "secret-key" --secret

# Update variable
supercheck variables update --key "API_URL" --value "https://api-staging.example.com"

# Delete variable
supercheck variables delete --key "API_URL"
```

#### Status Page Management

```bash
# List all status pages
supercheck status-pages list

# Create status page
supercheck status-pages create --name "Service Status" --url "status.example.com"

# Update status page
supercheck status-pages update --id "status-123456789" --name "Updated Status"

# Delete status page
supercheck status-pages delete --id "status-123456789"
```

### API Commands

#### Authentication

```bash
# Generate API key
curl -X POST "https://api.supercheck.io/v1/auth/api-keys" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "My API Key",
    "scopes": ["tests:read", "tests:write", "jobs:execute"]
  }'

# List API keys
curl -X GET "https://api.supercheck.io/v1/auth/api-keys" \
  -H "Authorization: Bearer YOUR_API_KEY"

# Revoke API key
curl -X DELETE "https://api.supercheck.io/v1/auth/api-keys/{keyId}" \
  -H "Authorization: Bearer YOUR_API_KEY"
```

#### Test Management

```bash
# List tests
curl -X GET "https://api.supercheck.io/v1/tests" \
  -H "Authorization: Bearer YOUR_API_KEY"

# Create test
curl -X POST "https://api.supercheck.io/v1/tests" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "title": "API Test",
    "type": "api_test",
    "script": "const { test, expect } = require(\"@playwright/test\");\n\ntest(\"API test\", async () => {\n  console.log(\"Test passed\");\n});"
  }'

# Run test
curl -X POST "https://api.supercheck.io/v1/tests/{testId}/run" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "environment": "staging",
    "variables": {
      "API_BASE_URL": "https://api-staging.example.com"
    }
  }'

# Get test results
curl -X GET "https://api.supercheck.io/v1/tests/{testId}/results/{executionId}" \
  -H "Authorization: Bearer YOUR_API_KEY"
```

## Configuration Examples

### Test Configuration

#### Browser Test Configuration

```javascript
// test-config.js
module.exports = {
  title: "User Login Test",
  description: "Test user login functionality",
  type: "browser_test",
  environment: "staging",
  timeout: 30000,
  retries: 2,
  tags: ["authentication", "critical"],
  priority: "high",
  script: `
    const { test, expect } = require("@playwright/test");

    test("user can login", async ({ page }) => {
      await page.goto("${API_BASE_URL}/login");
      await page.fill('[name="username"]', "testuser");
      await page.fill('[name="password"]', "password123");
      await page.click('[type="submit"]');
      
      await expect(page.locator(".welcome-message")).toBeVisible();
    });
  `,
  variables: {
    API_BASE_URL: "https://staging.example.com",
    USERNAME: "testuser",
    PASSWORD: "password123",
  },
};
```

#### API Test Configuration

```javascript
// api-test-config.js
module.exports = {
  title: "API Health Check",
  description: "Test API health endpoint",
  type: "api_test",
  environment: "staging",
  timeout: 15000,
  retries: 1,
  tags: ["api", "health"],
  priority: "high",
  script: `
    const { test, expect } = require("@playwright/test");

    test("API health check", async ({ request }) => {
      const response = await request.get("${API_BASE_URL}/health", {
        headers: {
          "Authorization": "Bearer ${API_KEY}",
          "Content-Type": "application/json"
        }
      });

      expect(response.status()).toBe(200);
      
      const data = await response.json();
      expect(data.status).toBe("healthy");
      expect(data.timestamp).toBeDefined();
    });
  `,
  variables: {
    API_BASE_URL: "https://api-staging.example.com",
    API_KEY: "staging-api-key",
  },
};
```

## Troubleshooting Tips

### Common Issues and Solutions

#### Test Execution Issues

**Issue: Test fails with timeout error**

```bash
# Solution: Increase timeout
curl -X PUT "https://api.supercheck.io/v1/tests/{testId}" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "timeout": 60000
  }'
```

**Issue: Test fails with authentication error**

```bash
# Solution: Check API key and permissions
curl -X GET "https://api.supercheck.io/v1/auth/api-keys" \
  -H "Authorization: Bearer YOUR_API_KEY"

# Update API key with correct scopes
curl -X PUT "https://api.supercheck.io/v1/auth/api-keys/{keyId}" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "scopes": ["tests:read", "tests:write", "jobs:execute"]
  }'
```

**Issue: Test fails with variable resolution error**

```bash
# Solution: Check variable exists and is accessible
curl -X GET "https://api.supercheck.io/v1/variables" \
  -H "Authorization: Bearer YOUR_API_KEY"

# Create missing variable
curl -X POST "https://api.supercheck.io/v1/variables" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "key": "MISSING_VARIABLE",
    "value": "variable-value",
    "type": "regular"
  }'
```

#### Job Execution Issues

**Issue: Job fails with queue full error**

```bash
# Solution: Check queue status and capacity
curl -X GET "https://api.supercheck.io/v1/status" \
  -H "Authorization: Bearer YOUR_API_KEY"

# Wait for queue to clear and retry
curl -X POST "https://api.supercheck.io/v1/jobs/{jobId}/run" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "options": {
      "retry": true,
      "retryDelay": 60000
    }
  }'
```

**Issue: Job fails with variable resolution error**

```bash
# Solution: Check job variables
curl -X GET "https://api.supercheck.io/v1/jobs/{jobId}" \
  -H "Authorization: Bearer YOUR_API_KEY"

# Update job with correct variables
curl -X PUT "https://api.supercheck.io/v1/jobs/{jobId}" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "variables": {
      "API_BASE_URL": "https://api-staging.example.com",
      "API_KEY": "staging-api-key"
    }
  }'
```

  /^https?:\/\/(www\.)?[-a-zA-Z0-9@:%._\+~#=]{1,256}\.[a-zA-Z0-9()]{1,6}\b([-a-zA-Z0-9()@:%_\+.~#?&//=]*)$/;
